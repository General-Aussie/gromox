邮件地址列表分为3种类型
  1. 普通地址列表

    2. 普通地址列表需要手工添加或删除成员，其中的地址成员可以是域内地址，也可以是外域地址。当添加该类型的邮件地址列表成功以后请点击条目操作栏中的“展开编辑”进行成员的添加与删除
  2. 部门地址列表

    3. 部门地址列表包含整个部门的成员地址，这种类型的列表不需要手工进行成员编辑。需要注意的是，部门地址列表以部门代号为邮件地址，所以如果要添加部门地址列表，请保证部门代号不被其它用户名占用
  3. 整域地址列表

    4. 整域地址列表就是域内的完整成员地址列表，和部门地址列表一样，该列表也不需亚对成员进行手工编辑。列表所使用的邮件地址可为任意地址
每个邮件地址列表都规定了展开权限，也就是向邮件列表地址发送邮件的发件人需要满足的一定的条件地址列表才能被完整展开
  1. 完全公开

    2. 任何发件人都可以向该邮件列表地址发送邮件，邮件地址列表都将被正常展开
  2. 内部成员

    3. 只有发件人是内部成员时，邮件地址列表才能被正常展开，其它情况下将产生退信给发件人
  3. 同域地址

    4. 当发件人使用的域名与与邮件列表使用的域相同时邮件地址列表被正常展开，否则将产生退信给发件人
  4. 指定展开发件人

    5. 只有被指定的发件人才可以展开邮件列表，否则将产生退信给发件人
另外需要注意的是，普通邮件地址列表的总数限制为20个，其每个普通邮件地址列表所能容纳的邮件地址最大数量为200个
